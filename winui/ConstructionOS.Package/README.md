# ACO — MSIX packaging project (U6, finish in VS 2022)

This folder holds the **pre-filled `Package.appxmanifest`** for the Windows
Application Packaging Project that wraps the WinUI 3 client + the PyInstaller
backend sidecar into a signed MSIX.

**It is a scaffold.** The `.wapproj` and the `Images\` visual assets are
generated by Visual Studio and are **not** checked in here (a hand-written
`.wapproj` is version-specific and a mismatched one wastes time; asset PNGs must
be the exact MSIX dimensions or the build fails). None of this has been built or
signed from the headless dev environment.

## What's already done (elsewhere in the repo)
- **Backend sidecar** — `winui/build-sidecar.ps1` freezes `web_main.py` into a
  standalone `ACO.Backend.exe` (no Python needed). ✅
- **Auto-launch** — `Services/BackendLauncher.cs` starts `Backend\ACO.Backend.exe`
  (next to the app) when the port is free. ✅ Verified end-to-end.

## Finish it in VS 2022 (Windows box)
1. **Build the sidecar:** `pwsh winui/build-sidecar.ps1` → produces
   `ACO.Backend.exe`.
2. **Add the packaging project:** open the solution → *Add → New Project →
   Windows Application Packaging Project* → name it `ConstructionOS.Package` here.
3. **Reference the app:** in the packaging project, *Add → Project Reference →
   `ConstructionOS.WinUI`*, and set it as the **Entry Point** (Applications node).
4. **Replace the manifest:** overwrite the generated `Package.appxmanifest` with
   the one in this folder, then set `Identity/@Publisher` to your signing cert's
   subject. (Keep `rescap:runFullTrust` — the app launches the localhost sidecar.)
5. **Bundle the sidecar:** add `ACO.Backend.exe` to the packaging project under a
   `Backend\` folder with *Package Action = Content* (so it lands at
   `Backend\ACO.Backend.exe` next to the app exe, where `BackendLauncher` looks).
6. **Visual assets:** in the manifest designer → *Visual Assets* → generate all
   tiles/logos from `construction_app/resources/logo_square.png` (Radiant Orange).
7. **Package + sign:** *Publish → Create App Packages* → sideload → sign with your
   cert (or self-signed for testing; install the cert into *Trusted People*).

## Constraints (unchanged)
- Backend listens on **127.0.0.1 only**.
- No business maths in C# — the client calls `/api/*`.
- The one client settings file: `%LOCALAPPDATA%\ACO\winui-settings.json`.
